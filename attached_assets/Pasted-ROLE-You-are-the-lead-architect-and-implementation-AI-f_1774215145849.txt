AUDIO PIPELINE TO IMPLEMENT

PIPELINE STAGE 1 — INGESTION
Build a production-grade ingestion pipeline:
- accept wav/mp3/flac/m4a/video input
- transcode to canonical wav
- validate duration/sample rate/channels
- loudness normalize
- store source and canonical asset
- compute checksum for deduplication

Output:
{
  "assetId": "...",
  "durationSec": ...,
  "sampleRate": 44100,
  "channels": 2,
  "format": "wav",
  "storageUrl": "...",
  "checksum": "..."
}

PIPELINE STAGE 2 — FEATURE EXTRACTION

- onset envelope
- beat positions
- tempo curve
- meter estimate
- chroma / HPCP
- tonal centroid
- spectral contrast
- RMS / energy curve
- section boundaries
- vocal activity estimate
- melody contour

Cache extracted features so they are reusable across jobs.

PIPELINE STAGE 3 — RHYTHM / TEMPO / METER
Implement:

- downbeat approximation
- bar segmentation

Return:
- global tempo

- beat grid
- bar grid
- confidence
- warnings

PIPELINE STAGE 4 — KEY / MODE / MODULATION

Return:
- global key

- modulation events
- alternatives
- confidence
- evidence summary

PIPELINE STAGE 5 — CHORD RECOGNITION
Replace pure template matching with a hybrid approach:

- expose editable note events to frontend

Return:
